# Домашнее задание к занятию «Введение в микросервисы» - Крюков Егор

## Задача

Руководство крупного интернет-магазина, у которого постоянно растёт пользовательская база и количество заказов, рассматривает возможность переделки своей внутренней   ИТ-системы на основе микросервисов. 

Вас пригласили в качестве консультанта для оценки целесообразности перехода на микросервисную архитектуру. 

Опишите, какие выгоды может получить компания от перехода на микросервисную архитектуру и какие проблемы нужно решить в первую очередь.


## Ответ

## Выгоды микросервисной архитектуры

1. **Масштабируемость** - Можно отдельно масштабировать нагруженные части системы (заказы, платежи), а не весь сервис целиком.

2. **Отказоустойчивость** - Сбой одного сервиса не приводит к падению всей системы.

3. **Быстрее разработка** - Команды могут работать независимо над разными сервисами.

4. **Гибкость технологий** - Для каждого сервиса можно использовать подходящий стек технологий.

5. **Упрощение поддержки** - Маленькие сервисы проще обновлять и сопровождать.

---

## Основные проблемы при переходе

1. **Сложность архитектуры** - Микросервисы сложнее монолита, требуется грамотное проектирование.

2. **Инфраструктура** - Нужны Docker, CI/CD, мониторинг и оркестрация.

3. **Работа с данными** - Нет общих транзакций, нужно учитывать согласованность данных.

4. **Мониторинг и логи** - Без централизованного мониторинга сложно искать ошибки.

5. **Подготовка команды** - Требуется опыт и изменение процессов разработки.
